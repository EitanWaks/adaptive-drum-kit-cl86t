================================================================================
                    WIRING DIAGRAM SUMMARY - CL86T MOTOR CONTROL
================================================================================

COMPONENTS:
  - Arduino UNO R3
  - CL86T Closed-Loop Stepper Driver
  - 34E1K-120 Stepper Motor (NEMA 34)
  - LE-350-60 Power Supply (60V, 5.8A)

================================================================================
ARDUINO TO CL86T DRIVER (Control Signals)
================================================================================

CL86T Control Terminals:
  PUL+, PUL-      (Pulse/Step signal)
  DIR+, DIR-      (Direction signal)
  ENA+, ENA-      (Enable signal, optional)

Connection:
Arduino Pin 2  ──────────────>  CL86T PUL+  (Step Pulse)
Arduino Pin 3  ──────────────>  CL86T DIR+  (Direction)
Arduino Pin 4  ──────────────>  CL86T ENA+  (Enable, optional)
Arduino GND    ──────────────>  CL86T PUL-  (Ground)
Arduino GND    ──────────────>  CL86T DIR-  (Ground)
Arduino GND    ──────────────>  CL86T ENA-  (Ground)

Note: Can use single GND wire with multiple connections

================================================================================
POWER SUPPLY TO CL86T DRIVER
================================================================================

LE-350-60 Terminals:
  +V, +V, +V  (three positive - use any one)
  -V, -V, -V  (three negative - use any one)
  GND         (ground/frame ground)
  N           (neutral - AC input)
  L           (line/live - AC input)

AC Input Wire Colors:
  Blue   → L   (line/live)
  Brown  → N   (neutral)
  Yellow → GND (ground)

CL86T Power Input:
  AC, AC      (two AC terminals - accept AC or DC, no polarity for DC)

Connection:
LE-350-60 +V (any one)  ──────────>  CL86T AC (first terminal)
LE-350-60 -V (any one)  ──────────>  CL86T AC (second terminal)
LE-350-60 GND           ──────────>  (Frame ground, optional)

WARNING: Check polarity! Reversed polarity will damage driver.
Use 14-16 AWG wire for power connections.

================================================================================
MOTOR TO CL86T DRIVER
================================================================================

✅ VERIFIED CONFIGURATION:

34E1K-120 Black   ────────────>  CL86T A+  (Phase A Positive)
34E1K-120 Green   ────────────>  CL86T A-  (Phase A Negative)
34E1K-120 Red     ────────────>  CL86T B+  (Phase B Positive)
34E1K-120 Blue    ────────────>  CL86T B-  (Phase B Negative)

Color Code (Verified):
  A+: Black
  A-: Green
  B+: Red
  B-: Blue

If motor vibrates, swap A+ with A- (Black ↔ Green) or B+ with B- (Red ↔ Blue)

================================================================================
POWER-ON SEQUENCE
================================================================================

1. Ensure motor shaft is free to rotate
2. Connect Arduino USB to computer
3. Open serial monitor (115200 baud)
4. Plug in power supply AC input
5. Observe power supply LED/indicator
6. Send '?' command to verify communication
7. Send 'R' command to test one revolution

================================================================================
SAFETY WARNINGS
================================================================================

⚠️  HIGH VOLTAGE: 60V DC can be dangerous
⚠️  Always disconnect power before making connections
⚠️  Double-check all connections before powering on
⚠️  Use proper wire gauge (14-16 AWG for power, 22-24 AWG for signals)
⚠️  Ensure proper grounding
⚠️  Keep work area dry and free of conductive materials
⚠️  Never touch live connections while power is applied

================================================================================
TROUBLESHOOTING QUICK REFERENCE
================================================================================

Motor doesn't move:
  - Check power supply is on
  - Verify enable signal (try 'E' command)
  - Check all connections
  - Verify motor phase wiring

Motor vibrates:
  - Swap A+ and A- wires
  - Or swap B+ and B- wires

Wrong direction:
  - Change DIR_INVERT to true in pins.h
  - Or swap DIR+ and DIR- connections

Arduino not responding:
  - Check USB cable
  - Verify COM port
  - Check serial monitor baud rate (115200)
